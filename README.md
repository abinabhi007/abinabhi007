## Engineering Philosophy

> *"A component is not done when it works. It's done when the next developer can understand, extend, and delete it without asking anyone."*

- **Ship with evidence** — every decision I make should be traceable to a metric or a user need, not a preference
- **Constraints are creative** — some of my best engineering came from working within free-tier limits, not around them
- **Ownership over credit** — I'd rather own a small, live product than contribute to a large, unshipped one
- **Readable > clever** — code is read 10× more than it's written; optimise for the reader
